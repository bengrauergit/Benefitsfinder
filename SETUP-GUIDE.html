BenefitsFinder Australia

Setup & Deployment Guide — Step by Step

📋 What you're setting up:

  • ✅ Supabase — free database to store user profiles and claim tracking
  • ✅ Vercel — free hosting to put your website live on the internet
  • ✅ GitHub — free code storage (required by Vercel) — takes 2 minutes

⏱️ Total time: approximately 20–30 minutes

1 Set Up Your Database (Supabase)

Step 1 — Create a free Supabase account

  1. Open a new browser tab and go to supabase.com
  2. Click the green "Start your project" button
  3. Sign up with your GitHub account, Google account, or email
  4. Once signed in, click "New Project"
  5. Choose "Free plan"
  6. Fill in: Project name (e.g. "benefits-finder"), a database password (save this!), and choose a region close to Australia (e.g. Southeast Asia (Singapore))
  7. Click "Create new project" — takes about 2 minutes to set up

Step 2 — Create the database tables

  1. In your Supabase project, look at the left sidebar and click "SQL Editor"
  2. Click "New query"
  3. Open the file schema.sql (in the same folder as this guide) in a text editor (Notepad on Windows, TextEdit on Mac)
  4. Select all the text (Ctrl+A on Windows, Cmd+A on Mac) and copy it
  5. Paste it into the Supabase SQL Editor
  6. Click the green "Run" button (or press Ctrl+Enter)
  7. You should see a success message at the bottom

Step 3 — Copy your Supabase credentials (IMPORTANT!)

  1. In Supabase, click "Project Settings" in the left sidebar (the gear icon ⚙️)
  2. Click "API" in the settings menu
  3. You'll see two important values — copy them both into a temporary notepad:
    • Project URL — looks like: https://abcdefghij.supabase.co
    • anon / public key — a very long string starting with "eyJ..."
  4. ⚠️ Do NOT copy the "service_role" key — only the "anon public" key

Step 4 — Add your credentials to the app

  1. Open the file index.html (in the same folder as this guide) in a text editor
  2. Near the top of the file (around line 300), find these two lines: 
    const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
  3. Replace YOUR_SUPABASE_URL with your Project URL (keep the quotes)
  4. Replace YOUR_SUPABASE_ANON_KEY with your anon key (keep the quotes)
  5. Save the file (Ctrl+S on Windows, Cmd+S on Mac)

After editing, it should look like this:

const SUPABASE_URL = 'https://abcdefghij.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIs...(long string)';

Step 5 — Enable Email Magic Link sign-in

  1. In Supabase, click "Authentication" in the left sidebar
  2. Click "Providers"
  3. Make sure "Email" is enabled (it should be on by default)
  4. Ensure "Confirm email" is turned ON — this enables magic links
  5. Click "Save"

2 Put the Files on GitHub

Step 6 — Create a free GitHub account (if you don't have one)

  1. Go to github.com
  2. Click "Sign up" and create a free account
  3. Verify your email address

Step 7 — Upload your files to GitHub

  1. Once logged in to GitHub, click the "+" button in the top right corner
  2. Select "New repository"
  3. Name it: benefits-finder
  4. Make sure it's set to "Public"
  5. Click "Create repository"
  6. On the next page, click "uploading an existing file" link
  7. Drag and drop these 3 files from your computer:
    • index.html (the one you edited with your Supabase credentials)
    • schema.sql
    • vercel.json
  8. Click "Commit changes" (the green button at the bottom)

3 Deploy Your Website (Vercel)

Step 8 — Create a free Vercel account

  1. Go to vercel.com
  2. Click "Sign Up"
  3. Choose "Continue with GitHub" — this links your GitHub account automatically
  4. Follow the prompts to authorise Vercel

Step 9 — Import your project

  1. Once in Vercel, click "Add New Project"
  2. You'll see your GitHub repositories listed — click "Import" next to benefits-finder
  3. Leave all settings as default (Vercel will detect it's a static site)
  4. Click the big "Deploy" button
  5. Wait about 30 seconds — you'll see a success screen with confetti 🎉
  6. Your website URL will be shown — something like benefits-finder.vercel.app

Step 10 — Update Supabase with your website URL

This allows the email magic link to redirect back to your site.

  1. Copy your Vercel URL (e.g. https://benefits-finder.vercel.app)
  2. Go back to Supabase → Authentication → URL Configuration
  3. In "Site URL", enter your Vercel URL
  4. In "Redirect URLs", add your Vercel URL with /** at the end
  5. Click "Save"
🎉

You're Live!

Your BenefitsFinder website is now live on the internet. Anyone can visit your Vercel URL, complete the quiz, and track their benefits claims — with all data securely saved to your database.

To make changes: Edit index.html → upload to GitHub → Vercel auto-deploys in 30 seconds.
Troubleshooting: If magic links don't work, double-check your Supabase Site URL. If data doesn't save, ensure your SUPABASE_URL and SUPABASE_ANON_KEY are correctly entered in index.html. The app works in "demo mode" (saves to browser only) if Supabase is not configured.
